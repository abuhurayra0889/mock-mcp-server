# Mock MCP Server — Documentation

A mock external MCP server that simulates a third-party builder server on the CollabAI Marketplace. Used for end-to-end testing of the Control Tower MCP proxy Edge Function before the June 12 Dhaka Buildfest launch.

---

## Stack

- **Runtime:** Deno
- **Framework:** Hono v4.7.10
- **Production URL:** https://mock-mcp-server.abuhurayra.deno.net

---

## Project Structure

```
mock-mcp-server/
├── main.ts                  # App entry point, health check, route wiring
├── deno.json                # Deno tasks and import map
├── middleware/
│   └── auth.ts              # Bearer token validation middleware
└── routes/
    ├── run.ts               # POST /run — happy path
    └── scenarios.ts         # POST /run/timeout | /run/error | /run/bad-auth
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `EXPECTED_KEY` | Optional | When set, auth middleware validates the token against this exact value. If not set, only token format is checked. Set to the real `sk-collabai-<uuid>` generated by the approval flow during E2E testing. |

---

## Authentication

Every protected route requires a `Bearer` token in the `Authorization` header.

**Token format:** `sk-collabai-<uuid>`

**Validation logic:**
1. Header must be present and start with `Bearer `
2. Token must start with `sk-collabai-` and have characters after the prefix
3. If `EXPECTED_KEY` env var is set, token must match it exactly

**Failure response:**
```json
{ "error": "unauthorized", "detail": "<reason>" }
```
HTTP status: `401`

## Routes

### `GET /health`

Health check. No auth required.

**Response:**
```json
{ "status": "ok", "server": "mock-mcp-server" }
```

---

### `POST /run`

Happy path. Simulates a working builder MCP server.

**Headers:**

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer sk-collabai-<uuid>` |
| `Content-Type` | Yes | `application/json` |
| `x-ct-org-id` | No | Control Tower org UUID — echoed back in response for verification |

**Request body:**
```json
{
  "input": {
    "user_message": "Summarize this document",
    "context": { "doc_id": "abc123" }
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `input.user_message` | string | Yes | The message or task from the CT user |
| `input.context` | object | No | Arbitrary metadata (doc IDs, session info, etc.) |

**Success response (`200`):**
```json
{
  "status": "ok",
  "agent": "mock-summarizer-v1",
  "output": "Mock response for: \"Summarize this document\"",
  "tokens_used": 55,
  "echoed": {
    "ct_org_id": "org-abc-999",
    "received_context": { "doc_id": "abc123" }
  }
}
```

**Error responses:**

| Status | Condition |
|--------|-----------|
| `400` | `input.user_message` is missing or not a string |
| `401` | Token missing, wrong format, or mismatch |

---

### `POST /run/timeout`

Simulates a builder server that never responds in time. Sleeps for 35 seconds before replying.

**Purpose:** Verify the proxy sets a hard timeout, logs `success = false` in `agent_usage_log`, and returns a clean error to CT instead of hanging.

**Auth:** Required (same Bearer token)

---

### `POST /run/error`

Simulates a builder server that crashes with an internal error.

**Purpose:** Verify the proxy does not bubble up raw 500 details to CT clients.

**Auth:** Required (same Bearer token)

**Response (`500`):**
```json
{ "error": "internal_server_error", "detail": "Builder agent crashed" }
```

---

### `POST /run/bad-auth`

Always returns `401` regardless of the token provided.

**Purpose:** Test the proxy from the other direction — call the proxy with a wrong/missing key and verify `success = false` is logged and CT gets a clean unauthorized error.

**Auth:** Not checked (always rejects)

**Response (`401`):**
```json
{ "error": "unauthorized", "detail": "Invalid API key" }
```

## Running Locally

**Prerequisites:** Deno installed (`deno --version`)

**Dev mode (auto-restarts on file changes):**
```bash
EXPECTED_KEY=sk-collabai-test-1234 deno task dev
```

**Production mode:**
```bash
EXPECTED_KEY=sk-collabai-test-1234 deno task start
```

Server starts on `http://localhost:8000`.

## Local Test Commands

Replace `sk-collabai-test-1234` with whatever you set as `EXPECTED_KEY`.

```bash
# Health check
curl http://localhost:8000/health

# Happy path
curl -X POST http://localhost:8000/run \
  -H "Authorization: Bearer sk-collabai-test-1234" \
  -H "x-ct-org-id: org-abc-999" \
  -H "Content-Type: application/json" \
  -d '{"input": {"user_message": "Summarize this document", "context": {"doc_id": "123"}}}'

# Wrong token → 401
curl -X POST http://localhost:8000/run \
  -H "Authorization: Bearer sk-collabai-WRONG" \
  -H "Content-Type: application/json" \
  -d '{"input": {"user_message": "test"}}'

# Missing token → 401
curl -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"input": {"user_message": "test"}}'

# 500 error scenario
curl -X POST http://localhost:8000/run/error \
  -H "Authorization: Bearer sk-collabai-test-1234" \
  -H "Content-Type: application/json" \
  -d '{"input": {"user_message": "test"}}'

# Timeout scenario (killed after 5s by curl)
curl -X POST http://localhost:8000/run/timeout \
  -H "Authorization: Bearer sk-collabai-test-1234" \
  -H "Content-Type: application/json" \
  -d '{"input": {"user_message": "test"}}' \
  --max-time 5

# Bad input → 400
curl -X POST http://localhost:8000/run \
  -H "Authorization: Bearer sk-collabai-test-1234" \
  -H "Content-Type: application/json" \
  -d '{"input": {}}'
```

## Deploying to Deno Deploy

### Option A — GitHub auto-deploy (recommended)

1. Push this folder to a GitHub repository
2. Go to [deno.com/deploy](https://deno.com/deploy) → New Project → link the repo
3. Set entry point to `main.ts`
4. Add env var `EXPECTED_KEY=sk-collabai-<real-uuid>` in project settings
5. Every push to `main` triggers an automatic redeploy

### Option B — Manual deploy via CLI

```bash
deployctl deploy --project=mock-mcp-server main.ts
```

## E2E Test Checklist (against the full proxy flow)

- [ ] Insert row into `marketplace_agents` with `status = pending`, `endpoint_url = https://mock-mcp-server.abuhurayra.deno.net/run`
- [ ] Update `status = approved` → API key generator fires, `api_key_encrypted` populated
- [ ] Call `mcp-proxy` Edge Function with `agent_id` + test input → proxy hits `/run`, gets response
- [ ] Check `agent_usage_log` → row exists, `success = true`, `response_ms` is a real number
- [ ] Point `endpoint_url` to `/run/timeout`, call proxy → `success = false` in log, CT gets clean error
- [ ] Point `endpoint_url` to `/run/error`, call proxy → failure logged cleanly, no raw 500 to CT
- [ ] Call proxy with non-existent or `status = pending` `agent_id` → proxy rejects before calling mock
